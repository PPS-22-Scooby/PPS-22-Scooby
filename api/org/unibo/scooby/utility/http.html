trait Backend[R] extends HttpClient

Core of the HTTP logic. A HttpClient can mix-in a Backend to gain the corresponding, underlying implementation of HTTP calls. Each Backend must specify the type of the generated HTTP responses: it can also (and in fact should) be a type that does not depend on the underlying HTTP library.

Core of the HTTP logic. A HttpClient can mix-in a Backend to gain the corresponding, underlying implementation of HTTP calls. Each Backend must specify the type of the generated HTTP responses: it can also (and in fact should) be a type that does not depend on the underlying HTTP library.

Note that while the Request type is used inside all the backends, the response's one R is specific to the used Backend.

Type parameters

R

type of the HTTP responses generated by the Backend
